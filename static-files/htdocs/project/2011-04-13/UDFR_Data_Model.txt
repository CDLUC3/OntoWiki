    >isExtensionOf expresses a relationship between a format and its extension. For example, UTF-8 isExtensionOf ASCII since all valid ASCII byte streams can be used in the context of any UTF-8-aware process without any loss of ASCII-enabled semantic function. Using a valid UTF-8 byte stream in the content of an ASCII-only-aware process may result in some loss of UTF-8-enabled semantic function. The fact that format A is an extension of format B necessarily implies that format B is a restriction of format A, and vice versa. Thus, the choice of which relationship to use to define the associations is arbitrary. As a best practice, the one that is consistent with the temporal relationship of the associated formats should be used. Since PDF 1.4 was defined prior to PDF/A it makes better sense to say that PDF/A is a restriction of PDF 1.4.
